Instructions

To successfully compact your structure in to one structure block, make sure you have input the above fields correctly. The coordinate range should be Positive X to Positive Z. Meaning, all coordinate values in coord box 1 should be less then its corresponding value in coord box 2. Acceptable Example: Coord Box 1 = 1 1 1 and Coord Box 2 = 5 5 5. The structure name does not matter. Do not add any spaces in the structure name, or make it unnecessarily long.

As for the commands generated, you first want to make sure you are near the structure you are saving so all the chunks are loaded. Also, make sure you are high/low enough so that the commands generated do not spawn inside your structure (A displacement of 15 blocks or higher will work). First copy the 'Conflict Finder' command. Install it, and activate the chain that comes with it. This should report any conflicts that appear. What is a conflict? A conflict is a spot where a structure block component will be spawned, but a block of your structure will be erased if the installation were to take effect. If the chain does not report any errors, delete the chain and proceed to the next step.

Now, copy the Structure block Generator command. Paste it in, and you will see your structure mapped in the structure blocks. Activate the chain that spawns above you by turning it to always active. It should automatically dissapear upon activation. Proceed to the next step.

Finally, copy the Compacter command. Paste this in, and you will see a few command block machines spawned in. Turn on the repeating block that is on Needs Redstone. This should move the armorstand that is nearby. After the armorstand is killed, you can remove all the command blocks.

And then your structure is ready to be spawned in with a singular structure block. You can spawn it in with a structure block named "(structurename)1". Good luck! Any bugs should be reported to ThatOneTqnk.
